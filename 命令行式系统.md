# Mac操作逻辑

- 【常用工具】ripgrep - 快捷搜索、duti - 管理文件默认打开方式, iterm2

  ```
  duti -s com.microsoft.VSCode .json all  查看
  duti -x .json  设置
  ```

  

- 【文件目录交互】mac隐藏了文件夹根目录，在访达中一般只能直接交互式访问一些默认文件夹，一般在用户目录下的一级目录。如果需要扩展，只能自己手动收藏，类似不提供整个系统的文件树，但允许用户建立自己的文件树系统。**为了兼容命令行的全局文件操作，mac提供了搜索功能，可以全局搜索特定的文件夹，用于临时交互，希望长期交互的文件夹可以将其拖入个人收藏，或者以标签分类。**

- 【复制粘贴、触摸板】mac用command键执行复制粘贴，触摸板有两级压力感应，一般只适用第一级的压力反馈表达鼠标左键与右键。

- 【应用菜单栏】在mac上所有的应用都做了适配，其菜单栏放在整个屏幕的右上角。右上角第一个标记是iphone本机的标记，第二个标记则是该应用程序自己的标记，点击一般会带有一些设置等内容。

- 【快捷键】Control + Command + Q 锁屏、Shift-Command-4截屏（如果按住ctrl截图，则自动复制到剪切板）、Command+Ctrl+f进入/退出全屏。Command+W是关闭窗口，Command+M是将窗口最小化至程序坞。

  - 【snippet】command+shift+L打开snippetLibrary

  - 【undo/redo】command+z, command+shift+z

  - 【查看文件大小】du -h filename

    ```cpp
    使用--max-depth参数指定路径层级(但是mac不支持max-depth, macOS中直接使用-d参数 du -h -d 1 ./)
    -h表示自动转换单位，-h 是 "human‑readable"
    要查完后直接排序，可以用du -h -d 1 ./ | sort -rh  // -r为反向排序，-h是人类可读的数字排序
    ```

- 【stat信息】

  ```cpp
  16777229 17192029 -rw-r--r-- 1 zhoukailei staff 0 32428 
  "Dec  1 10:58:00 2025" "Dec  1 10:58:00 2025" "Dec  1 10:58:00 2025" "Dec  1 10:52:50 2025" 
  4096 64 0 ./lane/lane_navi_config.bin
    
  16777229 → 设备号 (device number) 表示文件所在的物理磁盘或分区的标识。
  17192029 → inode 号 文件在文件系统中的唯一标识，不管文件名怎么改，inode 不变。
  -rw-r--r-- → 文件权限
  1 → 硬链接数量 表示有多少个文件名指向同一个 inode。通常普通文件是 1。
  zhoukailei → 文件所有者用户名
  staff → 文件所属用户组
  0 → 文件的扩展属性标志位（常为 0，表示没有特殊标志）
  32428 → 文件大小（bytes 字节）
  "Dec 1 10:58:00 2025" （第一个时间）→ 最后访问时间 (Access time)
  "Dec 1 10:58:00 2025" （第二个时间） → 最后修改时间 (Modify time)
  "Dec 1 10:58:00 2025" （第三个时间） → 状态改变时间 (Change time)
  "Dec 1 10:52:50 2025" → 文件创建时间 (Birth time)
  4096 → 文件占用的磁盘空间（Bytes on disk）：按文件系统的最小分配单位计算，一般是 4KB。
  64 → 磁盘块数量（每块 512 字节）：64 × 512 = 32768 字节，这和文件大小接近。
  0 → 文件标志（flags），一般 0 表示没有特殊标志。
  ./lane/lane_navi_config.bin → 文件路径和名称
  ```

# 需求导向

- 如何在Linux中创建文件

  - touch命令：更新现有文件/目录/创建新文件

    touch file1.txt

    创建多个文件touch file1.txt file2.txt file3.txt

  - 使用重定向运算符创建文件。> 操作符将覆盖现有文件，而>>操作符将追加输出到文件中。**要创建一个空的零长度文件，只需要在定向操作符之前指定要创建的文件名即可**

    \> file1.txt

  - 使用cat命令创建文件，该命令用于读取和连接文件，但也可以用于创建新文件

    要创建新文件，请运行cat命令，后跟重定向操作符`>`和要创建的文件名。按`Enter`输入文字，完成后按`CRTL+D`保存文件。

    cat > file1.txt

  - 使用echo命令，该命令传递字符串到文件

    echo > file1.txt

- 如何在vim中进行多行删除
  - 首先在命令模式下，输入":set nu"显示行号；
  - 通过行号确定要删除的行
  - 命令输入":  32, 65d", 回车键，即可删除32-65行的文件。如果无意中删除错了，可以使用u键进行恢复
  - **dd命令可以删除光标所在行，删除光标所在以下的N行可以用Ndd**

- vim将光标移到指定行，再命令模式输入:n即可移动到第n行

- 只知道文件名或文件内容中包含某些特定字符串，但不知道具体的文件名。**==需要根据“特定字符串”反向查找文件==**

  - 查找**==文件名==**中包含**特定字符**的文件

    > 使用： **find 目录 -name \'\*特定字符串\*\'** 
    >
    > 如：
    >
    > ​		find ./ -name '\*2021-11-01\*'

  - 查找目录下**==文件内容==**包含特定字符的所有文件

    - 使用grep

      grep指令用于查找内容包含指定的范本样式的文件，如果发现某文件里的内容符合所指定的范本样式，预设grep指令会把含有范本样式的那一列显示出来。若不指定任何文件名称，或给与的文件名为-，则grep指令会标准输入设备读取数据

      > **标准格式**：grep -rn "特定字符串" ./
      >
      > 如： grep -rn ”set“ *
      >
      > 说明：
      >
      > -r: 递归查找
      >
      > -n:显示行号
      >
      > -R：查找所有文件包含子目录
      >
      > -i：忽略大小写
      >
      > *：表示当前目录所有文件，也可以是某个文件名

- netstat -ntp, 网络状态，-n不显示localhost只显示ip，-t表示tcp, -p表示进程号processId

- 【cat与vim】cat是concatenate的缩写，表示连接并将内容输出到控制台，最直观的用法是`cat file1 file2 > newfile`，即将两个或多个文件的内容连接起来形成一个新文件。在这种语义下，也可以用作查看、新建文件之类， `cat file`可以直接用来将file输出到控制台。`cat > file.txt`表示新建一个文件（>表示重定向）。

- 【将A文件夹内东西拷贝到B文件夹】

  ```cpp
  cp -r /path/to/A/* /path/to/B/
  
  -r 表示递归复制目录
  如果目标路径里已存在同名文件或目录，会覆盖旧内容（可以加 -i 提示确认）
  想要保留文件属性（时间戳、权限等），用：
  cp -a /path/to/A/* /path/to/B/
  -a 等价于 --archive，会保留所有属性，适合做目录备份。
  ```


- 将A文件夹移动到B文件夹，mv 源 目标为一下几种情况：

  ```cpp
  mv A B/  #这个表示将A文件夹及里面的内容移动到B/文件夹下，这里的参数都是真实的操作叶子节点
  ```

  - 目标已存在的目录 `mv A B`，如果B是目录，结果就是B/A。
  - 如果目标是不存在的目录或文件，则把源改名为目标。 `mv A B`。**此时不移动到里面**
  - 多个源文件时，目标必须是目录。`mv file1 file2 B/`

- 【定义自己的命令集】

  - 可以在HOME目录下mkdir一个bin目录。将该目录放在./\_zshrc下。

  - 在bin目录下vim一个文件，给他x(执行)权限。

  - 举个例子abt27

    ```cpp
    myfile
    	export PATH="/Library/Frameworks/Python.framework/Versions/2.7/bin:$PATH"
    	exec abt "$@"
    chmod +x myfile 对所有用户添加x权限
    
    ```

# Shell命令

- 运行Shell的.sh文件可以 sh xxx.sh

- 在类 Unix shell（如 bash、sh、zsh）里，export 是用来把一个 shell 变量标记为环境变量（environment variable）的命令。被 export 的变量会传递给该 shell 启动的子进程，而未 export 的普通变量只在当前 shell 里可见。

  ~~~cpp
  定义并导出：export VAR=value
  例如：
  export ASAN_OPTIONS="log_path=./asan_log:verbosity=1"
  ~~~
