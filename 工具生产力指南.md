## DOS(**Disk Operating System**)批处理脚本

- @echo off的意思是**在批处理运行命令的时候不会一条一条的显示执行的命令**，与之相匹配的还有@echo on。 当不想批处理文件中的命令一行一行在DOS中显示出来时（因为全部显示出来的话看起来会变得很杂乱），可以使用@echo off，将其放于程序的最上方就行了。

- setlocal enabledelayedexpansion 延迟环境变量扩展。

  - cmd在解释我们的命令时，首先会读取命令行一条完整的指令，看输入的命令格式是否符合要求，如果需要在命令中引入一条变量name, 可以在name的两侧各加上%, 形成%name%，然后cmd就会找到变量名对应的值（如果变量名不存在值，就返回空值），这个过程叫做**变量扩展**。

  - 在批处理命令执行时，可以大致分为两个阶段，解析阶段和执行阶段。解析阶段只会分析命令的语法结构，并将变量进行即时扩展（如果没有开延迟扩展命令），他不会真正执行命令，只会准备每一行命令（有点像编译），在这个过程中，只支持对变量进行第一步的初始化，而不支持随后的赋值操作（需要在命令执行后进行）。

  - 因此对于嵌套的情况，如下：

    ~~~script
    @echo off
    set var = test
    for %%i in (1, 1, 5) do (
    	set var = %%i
    	echo %var%
    )
    // 由于变量var在解析阶段已经被替换为test，因此在执行时，会输出5个test。
    ~~~

  - 对于这种循环或者其他需要将变量动态赋值的批处理指令，需要设置setlocal ENABLEDELAYEDEXPANSION,启动延迟命令扩展，将命令扩展放在执行阶段。**另外需要注意的是，启用delayedExpansion后，cmd会使用!xxx!判断是否为一个变量。**==%%修饰的是循环变量==
  - **同时，对于for循环语句中，在cmd窗口中的for循环语句为`for %variable in (set) do command [command-parameters]`。因此在批处理指令中，需要额外再加一个%进行标识原始字符。**]

- 在批处理脚本中，提取文件时for %%f in *.vcx.proj中，希望过滤出文件路径的文件名可以用`%%~nf`，其中%%为变量标识前缀，~为修饰符，表示获取文件的**文件名部分**（不包括扩展名）。如果有路径，`~n` 会去掉路径部分，只留下文件名。n为可选参数，表示要绝对路径的文件名name。

- ==注意只有循环变量有%%--两个百分号==

- 如果写好批处理脚本后，点击闪一下就没反应，一般是解析阶段发现语法错误。

- 【esc，颜色控制】esc（ASCII值为27）或者\033，在 C 语言、Bash、Python 等很多环境中，`\033` 是转义序列的开始符号，表示转义字符。它告诉终端接下来的字符序列是一个控制命令，而不是普通的文本。

  ~~~powershell
  echo esc[31mVisual Studio 2019 not found!Esc[0m
  
  // esc表示转义序列开始，[31m是颜色代码，表示接下来的字符显示红色，\033[0m 是重置格式的转义序列，0m 代表恢复默认的颜色和格式。
  \033[31m：红色文本
  \033[32m：绿色文本
  \033[33m：黄色文本
  \033[34m：蓝色文本
  \033[0m：重置颜色和格式（恢复默认）
  ~~~

- 【相对路径宏、\~%dp0】相对路径宏是提升脚本移植兼容性的重要工具，其中\~%0表示获取当前批处理脚本的路径。d与p是脚本中**参数扩展**部分，d表示driver，p表示path。0是一个参数编号，指代批处理文件本身。批处理脚本中的 `%0` 是指脚本文件的完整路径，而在%前面加上\~表示需要**启动参数扩展**，\~%dp0就表示只需要驱动器和路径部分，不需要**文件名与扩展名**。

- 可以写一个批处理脚本批量修改工程中所有模块的优化选项。

  ~~~cmd
  // 脚本模板
  @echo off
  setlocal enabledelayedexpansion
  
  REM 设置要查找的优化设置
  set oldOptimization=<OldOptimizationSetting>
  set newOptimization=<NewOptimizationSetting>
  
  // REM 查找所有 .vcxproj 文件
  for /r %%f in (*.vcxproj) do (
      // REM 创建一个备份文件
      copy "%%f" "%%f.bak"
      
      // REM 使用 PowerShell 替换优化设置
      powershell -Command "(Get-Content -path '%%f') -replace '<Optimization>!oldOptimization!</Optimization>', '<Optimization>!newOptimization!</Optimization>' | Set-Content -path '%%f'"
  )
  
  endlocal
  echo Done!
  pause
  ~~~

- 在这个命令中，为了实现替换文件中的文本内容，使用了 PowerShell。这是因为批处理脚本的功能有限，而 PowerShell 提供了更强大和灵活的文本处理功能。

  具体来说，PowerShell 在文本处理方面具有以下优势：

  1. **强大的正则表达式支持**：PowerShell 的 `-replace` 操作符支持正则表达式替换，这使得文本替换更加灵活和精确。
  2. **支持多行文本操作**：PowerShell 可以轻松处理多行文本文件，包括读取、处理和写入操作。
  3. **更复杂的文本操作**：PowerShell 提供了丰富的文本处理命令和函数，可以实现更复杂的文本操作，如搜索、分割、合并等。
  4. **更好的性能和可靠性**：PowerShell 是由 Microsoft 官方支持的脚本语言，通常具有更好的性能和可靠性，特别是在处理大型文件时。

# UV

- UV是一个包管理器，新版的UV也可以自己下载并管理python版本（升级为了环境管理器），放在~/.local/share/uv/python/中，如果想要看uv管理了哪些python版本，需要自己到这个目录看下。
- UV作为包管理器的实现模型为：在项目目录下执行`uv venv --python=[版本号]`，会在该目录下构建一个.venv文件夹，里面存放了一些包内容，另外还有激活脚本.venv/bin/activate，里面会将环境变量替换为指定的版本号（==前提是你之前自己下载/基于uv下载了对应版本的python==）。采用`source .venv/bin/activate`即可激活对应环境。
- UV还有一个tool（工具）包的管理语义，基于`uv tool install xxx`下载的包，会被放在uv的工具目录`.local/share/uv/tools`中，然后这个里面的bin会被uv链接到全局的`.local/bin/`，这样可以全局运行。
- uv add是uv包管理器的一个命令，用于向项目添加依赖包并自动更新 `pyproject.toml` 文件。
- 基于`uv add`命令添加的包，依赖写在`pyproject.toml`和`uv.lock`中，而实际的包文件安装在`.venv下`。
  - uv环境可以使用`source .venv/bin/activate`激活为当前默认环境，使用`deactivate`解除激活。


- 【uv与jupyter】要使用交互式jupyter需要分别安装notebook和Jupyter-core。这样基于jupyter(主命令) notebook(子命令)可以运行交互式notebook，但是默认直接使用内部的kernel（他不像conda将所有虚拟环境全部配置kernel在里面了）。如果希望执行某个虚拟环境下的python，需要`source .venv/bin/activate`激活后，安装kernerl, 并基于`python -m ipykernel install --user --name aone-venv --display-name "AoneTools venv"`注册kernel。后面打开jupyter notebook时就可以切换对应的kernel。
- 【python的解释器与包】python的管理模型为各自的解释器持有其各自的第三方包，一般在bin/下放有解释器，在lib/版本/site-packages下放有包。因此只要解释器路径选择正确，包一般也能找到。对应到UV的应用场景，无论是命令行/IDE执行/jupyter，只要解释器查找路径/解释器路径/kernel路径选择正确即可。
- 有pyproject.toml就是uv管理了，但是如果想在当前目录下建立自己的虚拟环境，可以使用uv venv .venv构建虚拟环境，这样是默认以最先找到的python解释器为版本，如果想要指定版本可以uv venv .venv -p 3.10

# Playwright

- 使用`playwright codegen`可以生成Inspector窗口。

- 可以使用如下代码启动一个浏览器

  ```python
  from playwright.async_api import async_playwright, Page
  async with async_playwright() as p:
  	browser = await p.chromium.launch(headless=False) # 使用headless为关闭界面的后台模式，False开启界面
    page = await browser.new_page()
    
  # 如果要启动一个带缓存的浏览器
  browser = await p.chromium.launch_persistent_context(user_data_dir="", headless=False)
  ```

- playwright支持使用已有的用户数据目录，需要制定`--user-data-dir`【只有chromium支持，Firefox/WebKit不支持】

- playwright在goto某个url时，一般会发送很多request，可以抓取一些request截取返回的字段，调用大模型进行文本处理。

- playwright在启动浏览器，并打开一个new_page后，page提供了on方法，来允许注册事件监听器，当事件触发时调用callback。Playwright的Page对象有很多事件，监听协议为`page.on("event名称", 回调函数)` 如下：

  ```
  request, response, console, load, domcontentloaded, dialog, pageerror, close
  ```

- 一般使用playwright做信息提取时，尽量使用文字，因此可以自己先看下request的具体链接。

- 【前段网页url拼接】前段网页有时候真实url会被隐藏，而地址栏只会显示主域名。在自动化场景下不希望做很多交互操作，因此需要拼接后的完整url，如果网页有分享功能，可以点一下分享然后把链接拿下来看下拼接逻辑。

- 在goto时有可能会被浏览器重定向，此时load模式可能一直无法结束(因为goto的资源无法被加载)，可以使用"domcontentloaded"模式，可以忽略资源直接返回response，根据response的真实request，可以看最终被浏览器处理器后的真实url访问。

- 使用 `page.pause()`，它会自动打开 **Playwright Inspector** 并让脚本停在那儿等待你手动继续。

- 【playwright定位器调试】可以使用`PWDEBUG=1 uv run python -m AoneAgent.tools.kun_lun_downloader ...`这个命令，PWDEBUG=1会启动全程调试

- **在使用page进行事件监听时，事件类型决定参数类型**：
  - `page.on("request", callback)` - 回调函数接收 `Request` 对象
  - `page.on("response", callback)` - 回调函数接收 `Response` 对象
  - `page.on("load", callback)` - 回调函数不接收参数

  **launch.json**
  允许我们配置和保存调试设置的详细信息，在launch.json里每配置好一个启动项，在运行和调试下拉列表里就可以选择启动：

  允许我们定义一些自动执行的诸如构建，打包，测试或部署软件系统之类的任务。

- 【自定义编译配置】c_cpp_properties.json、tasks.json、launch.json 其中第一个是指定模块内外链接依赖关系，所用的标准（对应默认链接哪个标准库），以及默认编译选项\_debug等（只是定义一些默认配置）；tasks.json描述一些定制编译任务，通过配置task可以在命令行或者快捷键执行这些任务（执行编译的命令）；launch.json定义调试会话的细节，例如要调试的可执行文件路径、调试时使用的命令行参数、环境变量和其他调试选项等。
  
  - 总的来说，可以理解为task描述了编译行为，launch描述了运行行为，c_cpp_properties描述了构建脚本。【**251012理解：C++程序运行基本有：编译、链接和运行三个大块，都有其各自的配置，比如编译有优化程度、使用编译器类型（clang、LLVM、VS...）等配置；链接有项目内包含与导出、项目间依赖、使用std版本等；运行有启动文件路径、运行的命令行参数等。**】


- VSCode存在两种设置setting.json的方式

# ssh/http协议

- 仓库使用 SSH 协议进行访问，您不需要每次输入用户名和密码，而是需要配置 SSH 密钥。ssh协议就是把从属于个人的个人账号转为**账号可以授权公钥，机器提供私钥最后完成授权的流程**，用户可以在个人机器上访问自己有权限的任何内容（账号生成密钥是不可逆的，保障了安全性）。

- .ssh文件夹是SSH（Secure Shell）协议的一部分，该协议用于在网络上安全地进行远程登录和执行命令。在Windows操作系统的C盘下的隐藏文件夹.ssh中，有一个名为known_hosts的文件，这个文件的作用是存储远程主机的身份验证信息，以确保用户连接到远程主机时的安全性。

- 注意在执行ssh-keygen -t -rsa -C “邮箱”时，最好不要自己更改系统商城的默认名字.ssh/id_rsa，在进行与远程主机通信时，系统就是基于这个名字寻找的。

- **在生成密钥后会有两个文件，id_rsa.pub和id_rsa，前者是公钥，后者是私钥。我们会把公有密钥交给服务端，当需要从服务端请求内容的时候，要带上私有密钥。此时，服务器会通过一定的算法计算私有密钥，并判断计算的结果是否与公有密钥一样，如果不一样则响应请求失败。**

- **验证程序**：

  - 在使用ssh协议进行访问时，如果出错了，可以调用ssh -v+命令，debug访问过程，查看具体错误原因

  - 在使用 git 命令与服务端进行交互之前，我们可以先验证下服务器和客户端是否握手成功了

  ~~~ssh
  ssh -T git@xxx.com
  // 如果是 github，就可以直接写 git@github.com, 如果是 aaa.bbb.com，就可以写 git@aaa.bbb.com。 
  ~~~

  - ssh -t是SSH 客户端的一个选项。这个选项的作用是 **禁用伪终端分配**（disabling pseudo-terminal allocation）
  - 客户端通过ssh协议访问服务器（包括bitBucket, gitlab等）有些允许交互式访问（开一个命令行直接访问），有些只允许非交互式访问（只能通过git clone等命令）。
  - 在ssh -T的行为中有可能需要建立交互式会话，entering interactive session，从而导致失败，这并不是身份验证的问题
  - 对于不允许交互式访问的服务器，可以直接使用git clone命令查看ssh密钥有效性

- known_hosts文件
  - 当我们成功与服务端进行连接时，ssh 会记录服务端的 Host、IP 以及 rsa 文件。
  - 会随着验证慢慢增加
  - 如果手误点击了false，可以使用`ssh -T git@xxx.com`手动建立通信，更新kwnon_host文件。

- 【http拉包】如果使用http拉包需要验证http域名的账号密码，通常输入一次后可以保存在windows的凭据管理器（控制面板$\to$账户$\to$凭据管理器）中，但是http一般没有ssh快，且没有ssh安全。

- 【ssh与进程】在客户端发起连接请求时，ssh服务器会创建一个/多个进程处理这个连接，而**其后在shell中运行的所有任务默认都是该shell进程的子进程**，因此当ssh会话终止时，父进程断开，子进程会受到SIGHUP信号导致终止。因此在运行一些长耗时任务时，需要使用`nohup`, `screen或`tmux`等工具将任务放置在后台执行，避免挂在当前shell会话进程下。

# 正则表达式

- [一文搞懂正则表达式](https://zhuanlan.zhihu.com/p/655706221)
- ()表示分组，改变正则表达式中运算符的优先级，将整个组变为一个整体，便于应用量词（如*, +,?,{n}等）。

- ~~~cpp
  // 在正则表达式中，元字符 `?` 用来表示前一个字符是可选的。
  // 正则表达式为常用的字符集和常用的正则表达式提供了简写。\d表示匹配数字[0-9]。
  // 有一些常见的元字符，比如[]表示匹配方括号中的任意字符。
  // (xyz)字符组，按照确切的顺序匹配字符 xyz。[]字符类，匹配其中的任意字符。
  (-?[\d.]+),(-?[\d.]+)
  // 对于以上表达式，表示匹配()，()里面的有一个可选的-，接下来匹配[]中的数字或点，+表示数字或点有一个或多个
  ~~~

- 一般要表示以xx为后缀的，可以使用否定字符类+“加号”+后缀，比如

  ~~~cpp
  [^#]+# 表示所有以#结尾的字符串，+表示匹配前面的否定字符类一次或者多次。
  ~~~


- 如果要用正则精确匹配某一个单词的情况下，可以用\b表达单词边界。

  > `\b` 用于匹配单词边界（word boundary），即匹配单词与非单词字符之间的位置，它本身不匹配具体字符，而是匹配边界位置，用于精确匹配独立的单词，而不匹配其前后带字母、数字或下划线的情况。
  >
  > 比如\bRegion\b匹配Region。==要注意\b和\^/$的差别在于边界的定义，\b以空白为边界，寻找符合规则的单词；而\^/\$的语义为以换行符为边界，寻找符合规则的行。==

- .* 匹配任意字符，贪婪匹配

- 如果期望连续匹配多个有明确开始和结尾标识符的字符串，比如[]，最好用否定匹配字符类\[^xx\]，而不用任意字符类\.。

  ```cpp
  \[[^\]]*NaviSurfaceInfo[^\]]*\]
  ```

- 注意\.\*贪婪匹配默认不支持换行，如果要支持换行，可以(.|\\n)*这样子，另外，对于贪婪匹配，如果在后面加一个?，将让其变为惰性匹配。

- 【负向前瞻】匹配不以dumpLanePath开头的任意字符

  ```cpp
  (?!dumpLanePath)(.|\n)  // 注意?!...这种负向前瞻表示 当前位置的后面不能匹配，不消耗文本，只做检查
  ```

- 【非捕获分组】`(?:...)` 是 **非捕获分组（non‑capturing group）**，分组但不创建捕获组

  ```cpp
  dumpmapping(?:(?!dumpmapping)(.|\n))*?11262264115451658248
  ```

  只起到逻辑分组的作用，不作为字符类

# 内存泄露工具

- .NET framework

  - 一般内存泄露的分析需要安装windows的.NET framework。

- linux下做内存泄漏检测就用Valgrind。

  > 使用方式为将程序编译好，然后通过valgrind来启动程序。
  >
  > 一般启动命令：`valgrind --log-file=./valgrind_report.log --leak-check=full --show-leak-kinds=all --show-reachable=no --track-origins=yes ./`
  >
  > 其中\-\-log-file指定报告输出文件，
  >
  > \-\-track-origions=yes是否显示未定义的变量，在堆、栈中被定义没有被initialised的变量都被定义为origions，这个选项是默认关闭的。
  >
  > \-\-show-leak-kinds=all 这里可以支持的选项有[definite|possible]，一般只需要关注definite，possible是可能有泄漏但不绝对。all也打印了reachable的内存，一般就使用默认的，即明确泄漏和潜在泄露的
  >
  > -\-leak-check=full 当服务器退出时是否进行完整的内存泄漏检查，选项有no|summary|full

- Valgrind泄漏报告

  > - 泄漏报告中，被分为了definitely lost, indirectly lost和possibly lost，reachable等。前两者是直接确定丢失，以及因为直接丢弃的内存而被间接丢失的。潜在泄漏为指针发生了偏移，没有指向内存头，但也有可能为了实现内存对齐，故意额外申请内存，返回对齐后的内存地址。仍然可达表示程序终止时，有资源没有被释放，这在异常终止时是正常的，但是在正常终止时是异常的。
  > - Valgrind的输出会包含导致内存泄漏的源代码文件和行号，可以帮助快速定位问题代码，基本是一个栈帧表，每一个堆栈块表示一处泄漏。
  > - Valgrind提供了一些参数优化其输出，\-\-num-callers=50（指定报告中显示的调用栈帧数），\-\-error-limit=no（显示所有错误，而不是默认的限制数量）,\-\-log-file=filename将输出重定向到指定的日志文件，

- PPT的曲线工具实际上使用的是**三次贝塞尔曲线**（三阶）的组合，每一段曲线由两个锚点和两个控制点构成，因此其无法直接绘制五阶贝塞尔曲线，但是可以通过多个三阶贝塞尔拼接来模拟五阶曲线。用Inscape要用
